## Overview

Parkomat.Agent is a **Windows-only .NET 10 / C#** agent that runs on a PC at a parking
site. It reads a PLC over **Modbus-TCP** and relays the site's state to the cloud
(**HiveMQ**) over **MQTT**, through a local **Mosquitto** bridge.

End-to-end data flow:

```
PLC → (Modbus-TCP) → Agent Service → (MQTT localhost:1883) → Mosquitto → (bridge/TLS) → HiveMQ → server
```

## Solution layout

`Parkomat.Agent.slnx` contains three projects under `src/`:

| Project | TFM | Responsibility |
|---|---|---|
| **Parkomat.Agent.Core** | `net10.0` | Shared config + protocol. Referenced by both other projects. `AgentPaths`, `ConfigStore`, `SiteConfig`, `SiteState`, `ModeTranslator`, `Messages`. |
| **Parkomat.Agent.Service** | `net10.0` | The Windows Service — the background heart. `Worker.cs` loop; `PlcReader` (NModbus), `OperationDetector`, `MqttPublisher` (MQTTnet), `BridgeConfigWriter`. |
| **Parkomat.Agent.Tray** | `net10.0-windows` | WinForms tray app for managing the service. `TrayContext`, `ServiceManager`, `SettingsForm`, `RegistersForm`. |

## Build / run / publish

Run from the `Parkomat.Agent/` directory:

```sh
# Build the whole solution
dotnet build Parkomat.Agent.slnx

# Run the Service locally as a console app (dev; DOTNET_ENVIRONMENT=Development via launchSettings)
dotnet run --project src/Parkomat.Agent.Service

# Run the Tray app
dotnet run --project src/Parkomat.Agent.Tray

# Run tests
dotnet test Parkomat.Agent.slnx

# Publish — output paths MUST match what installer.iss expects (publish/service, publish/tray)
dotnet publish src/Parkomat.Agent.Service -c Release -o publish/service
dotnet publish src/Parkomat.Agent.Tray    -c Release -o publish/tray
```

Installer: **Inno Setup** compiles `installer.iss` → `installer-output/ParkomatAgentSetup.exe`.

Tests live in `tests/Parkomat.Agent.Service.Tests` (xUnit). Current coverage is the
`OperationDetector` state/operation logic. For behavior with no unit coverage (PLC I/O, MQTT),
verify by running the Service/Tray directly.

## Runtime data locations

All paths come from `AgentPaths` (`src/Parkomat.Agent.Core/Configuration/AgentPaths.cs`).
Base folder is `C:\ProgramData\Parkomat\Agent\` — shared so both the Service (runs as SYSTEM)
and the Tray (runs as the user) can read/write it:

- `config.json` — site + PLC + MQTT settings (`ConfigStore` / `SiteConfig`)
- `heartbeat` — unix-seconds timestamp written after each successful PLC read
- `bridge.conf` — Mosquitto bridge config, generated by `BridgeConfigWriter`
- `agent.db` — reserved (declared path; not yet used)

## Architecture & control flow

`Worker.cs` (`ExecuteAsync`) runs the main loop: **read PLC → feed the brain → publish**.

- **`OperationDetector`** is the pure "brain": it *decides* what to publish and never sends
  anything itself. It remembers the previous MODE in `_previousMode` and emits a `state`
  message when the translated state changes, plus `operation` start/end messages on
  entry/exit transitions.
- **PLC failure handling:** 10 consecutive failed reads → publish `state: error` **once**
  (guarded by `plcErrorReported`), reset when the PLC recovers.
- **Heartbeat:** written on every successful read. The Tray (`ServiceManager`) reads it back
  to distinguish "really working" (heartbeat fresh ≤ 10s → colored icon) from merely
  "process alive" (service running but no fresh heartbeat).

## MQTT / server contract (non-obvious — easy to break)

These are enforced by the server; changing them silently breaks ingestion:

- **Topics:** `sites/{siteId}/state` and `sites/{siteId}/operation`.
- **Timestamps are unix _seconds_, not milliseconds.**
- **`SiteState` JSON strings are exact and server-enforced:** `ready`, `operating`, `error`,
  `maintenance`, `no_comm` (see `SiteState.cs` `[JsonStringEnumMemberName]`). Do not rename.
- **Operation messages must include a valid `state`** or the server rejects them.
- **`user` (card number) must be `""`, never `null`** — it is part of the server's dedup key.
- **LWT:** on disconnect the broker publishes `no_comm` to the state topic on the agent's behalf.
- **The Service always connects to local Mosquitto (`localhost:1883`), never directly to
  HiveMQ.** The `MqttConfig` host/port/username/password feed the generated `bridge.conf`
  (`BridgeConfigWriter`) — they are *not* the client connection settings.
- **MODE → state mapping lives only in `ModeTranslator`:** `0`=maintenance, `1`=ready,
  `2`/`3`=operating, `4`=init (ignored, no state), `5`=error. For operations, `2`=entry,
  `3`=exit.

## Conventions

- Source comments are in **Hebrew** — match that when editing existing files.
- `Nullable` and `ImplicitUsings` are enabled across projects.
- **Atomic file writes** (write to `.tmp`, then `File.Move(..., overwrite: true)`) in
  `ConfigStore` and `BridgeConfigWriter` — preserve this pattern for any config the service
  writes, so a power loss mid-write can't corrupt the file.

## Deployment notes

`installer.iss` (run as admin) registers **two** Windows services:

- **`ParkomatAgent`** — this app; `start= auto`, auto-restart on crash.
- **`Mosquitto`** — installed via `nssm.exe`; depends on `ParkomatAgent` so `bridge.conf`
  exists before it starts.

The Tray app auto-starts per-user via an `HKLM\...\Run` registry value.
